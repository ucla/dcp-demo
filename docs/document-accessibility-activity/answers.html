Alice In Not Accessible-land

Summary

The Page is coded to simulate a screen reader experience on a document that is poorly marked up.

Questions

  • Question 1: What did the shrinking potion taste like?
    • "a sort of mixed flavour of cherry-tart, custard, pine-apple, roast turkey, toffee, and hot buttered toast"
  • Question 2: How short did the shrinking potion make Alice?
    • 10 inches.
  • Question 3: What flavor of marmalade did Alice see on the jar while she fell?
    • Orange.

Page Alterations

  • If a user revisits the questions, focus is reset to the beginning of a document.
  • Text is not visible unless it is actively being read

Challenges

Text not visible until you see it
  • This is not unlike a screen reader's behavior in a situation without structure. Without providing a table of contents or some kind of structure, a screen reader will have to navigate a page by tabbing forward and backwards.
Questions aren't in order
  • Without structure, navigating a large document is often a bi directional experience, where a user may not know where they are specifically in a document
Lots of tabbing
  • Screen readers rely on structure to navigate larger docuemnts quickly. For example, a user is able to retrieve all headers or links and move quickly between them
Illustrations aren't visible.
  • Images that aren't marked up correctly can be invisible to screen reader users. Images without proper alt text may not convey all of the information you desire the audience to get out of the image.
Focus resets when scrolled back to the top
  • In a sequential environment that lacks structure, without navigation a screen reader user would have to manually traverse the dom, so to review a question, they would need to tab back up, then back to their position
